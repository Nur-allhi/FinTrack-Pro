### Per-batch plan:

#### Batch 1 (2 remaining) — `fix/batch-1-sync-engine`
- **T-226 BUG-003**: In `pullChanges()`, when `accountIdMap.get()` returns undefined for a transaction's `account_id`, skip the record (filter it out) instead of leaving the untranslated numeric ID.
- **T-228 BUG-035**: Add a `_retryQueue` array in syncEngine. Records skipped due to untranslatable FKs go into this queue. On next sync cycle, retry them before processing new unsynced records.

#### Batch 2 (1 remaining) — `fix/batch-2-auth-service`
- **T-238 BUG-037**: In `apiFetch`, when `_guestMode` is true and the URL isn't `/api/auth/`, instead of silently returning a fake 401, return a response with a specific header or status that `useLocalData.fetchData` can detect to show a "sign in to sync" toast.

#### Batch 3 (1 remaining) — `fix/batch-3-cascade-deletes`
- **T-241 BUG-032**: In the group delete API route (`api/db/groups.ts`), before setting `parent_id = null`, check if there are child accounts. If so, return a flag/warning in the response. The frontend can show a confirmation dialog.

#### Batch 5 (1 remaining) — `fix/batch-5-local-db`
- **T-245 BUG-004**: In `emptyBin`, after calling `flushPending()`, wait for sync to complete by checking `isSyncing()` in a polling loop or by making `flushPending` return a promise that resolves when sync finishes.

#### Batch 6 (2 remaining) — `fix/batch-6-use-local-data`
- **T-251 BUG-013**: In `fetchData`, ensure members are fully processed and written to localDb BEFORE accounts are upserted. Currently they run in parallel (`Promise.all`). Change to sequential: process members first, then accounts.
- **T-253 BUG-006**: Add `useMemo` or stable reference for `fetchData` to prevent polling interval from recreating when `toast` changes.

#### Batch 7 (3 remaining) — `fix/batch-7-app-state`
